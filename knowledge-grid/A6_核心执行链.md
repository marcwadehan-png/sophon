## 核心摘要

### 这是什么
核心执行链是Somn处理请求的标准流程，从接收问题到输出解决方案的完整链路。

### 核心流程
1. **接收理解** → 2. **分析诊断** → 3. **策略生成** → 4. **方案评估** → 5. **输出执行**

## 详细流程

### 五步主链
```
Step 1: 接收理解
  ├─ 接收用户输入
  ├─ 意图识别
  ├─ 实体提取
  └─ 需求澄清

Step 2: 分析诊断
  ├─ 问题分类
  ├─ 原因分析
  ├─ 数据收集
  └─ 现状诊断

Step 3: 策略生成
  ├─ 方案构思
  ├─ 资源评估
  ├─ 风险分析
  └─ 策略选择

Step 4: 方案评估
  ├─ 多方案对比
  ├─ ROI分析
  ├─ 可行性验证
  └─ 优化调整

Step 5: 输出执行
  ├─ 方案输出
  ├─ 执行建议
  ├─ 监控指标
  └─ 反馈机制
```

### 并行化优化
```
可以并行执行的模块：
- 数据获取（多个数据源同时请求）
- 方案构思（多个方案同时设计）
- 风险评估（多个风险同时分析）

必须串行的模块：
- 理解 → 诊断（依赖）
- 诊断 → 策略（依赖）
- 策略 → 评估（依赖）
```
